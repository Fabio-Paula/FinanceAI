# Quick Commit Guide

## Using `/commit`

Type this in Claude Code whenever you want to commit:

```
/commit
```

That's it! Claude will automatically:
1. ✅ Analyze your staged changes
2. ✅ Generate a smart commit message
3. ✅ Ask for approval
4. ✅ Create the commit

## Step-by-Step Example

```bash
# 1. Make changes to your files
# ... edit files ...

# 2. Stage them
git add src/components/Button.tsx src/styles/button.css

# 3. In Claude Code, type:
/commit

# 4. Claude shows the proposed message:
#
# style(ui): improve button component styling and layout
#
# - Refine button styles with better padding and borders
# - Update component styling for consistency
# 
# Co-Authored-By: Claude Haiku 4.5 <noreply@anthropic.com>

# 5. Approve and Claude commits automatically!
```

## What `/commit` Does

| Step | Action |
|------|--------|
| 1 | Reads `git diff --cached` |
| 2 | Analyzes file patterns |
| 3 | Detects change type (feat, fix, style, etc.) |
| 4 | Generates descriptive message |
| 5 | Proposes commit |
| 6 | Waits for approval |
| 7 | Creates commit |

## Examples

### Example 1: Simple Style Change
```bash
git add src/styles/globals.css
# In Claude:
/commit

# Result:
# style(ui): improve global CSS styling
```

### Example 2: New Backend Feature
```bash
git add server/routes/new-feature.ts
# In Claude:
/commit

# Result:
# feat(backend): create new feature route handler
```

### Example 3: Bug Fix
```bash
git add src/lib/utils.ts
# In Claude:
/commit

# Result:
# fix(frontend): resolve calculation error in utility function
```

## Available Commands

| Command | What It Does |
|---------|------------|
| `/commit` | Create smart commit (this is what you want!) |
| `/commit-help` | Show help for commit command |
| `git-smart-commit` | Full skill with detailed options |

## Tips & Tricks

### ✅ DO:
- Stage related changes together
- Use `/commit` right after staging
- Review the proposed message
- Ask Claude to adjust if needed

### ❌ DON'T:
- Forget to `git add` first
- Commit unrelated changes together
- Force commit without reviewing

## Troubleshooting

**"No staged changes found"**
- Run `git add <files>` first

**"Incorrect scope"**
- That's ok, just tell Claude to change it
- Example: "Change scope to backend"

**"Want more detail?"**
- Tell Claude: "/commit but with more detail"
- Or: "add more technical explanation"
